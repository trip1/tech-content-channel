# Research: I Built a Tiny Home Server and Here Is Why

## Research goal
Create a simple, current, beginner-friendly video that shows a real workflow rather than repeating hype.

## Trend context
- AI agents, coding assistants, local AI, cybersecurity basics, homelab projects, and AI-powered consumer tools are high-interest tech topics.
- Viewers are overloaded with broad claims, so this video should win by showing a small real test, clear tradeoffs, and a practical recommendation.

## Sources / references to check before recording
- Official docs/product pages for any tools named in the video.
- Current pricing or free-tier pages, if the video compares tools or subscriptions.
- Current security/privacy documentation, if the video mentions data handling.
- Recent release notes or version pages for tools used on screen.
- Your own screen recording and verification output from the demo.

## Claims to verify
- Tool names, feature names, and UI labels are current.
- Any pricing/free-tier statement is true on recording day.
- Any security/privacy statement is phrased carefully and backed by official docs.
- The demo can be reproduced from a clean starting point.

## Demo constraints
Show the device, dashboard, one running service, basic network access, and power/noise considerations.

## Recording notes
- Capture versions and timestamps in a small notes file or terminal command.
- Prefer demo accounts, sample repos, and sanitized test data.
- Keep one real failure or limitation in the final edit if it helps viewers trust the recommendation.

## Caveats to say out loud
- This is a beginner-friendly first test, not a universal benchmark.
- Products and AI tools change quickly.
- The safest recommendation is based on the workflow shown, not on marketing claims.
