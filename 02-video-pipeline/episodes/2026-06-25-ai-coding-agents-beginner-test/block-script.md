# Block script: I Tried 3 AI Coding Agents So You Do Not Have To

## Block 1: Cold open — show the result/conflict
**Time:** 0:00-0:25

**On screen**
- Fast montage: final demo/result, one warning/catch, and the title card.

**Voiceover / talking head**
> Three agents, one tiny real-world task, and a simple scoreboard: who actually helps a beginner ship?
>
> In this video I am going to keep it simple: one practical demo, one honest limitation, and one recommendation you can actually use.

**Retention beat**
Start with the payoff and hint that there is a catch.

**Open loop**
Can this trend survive a simple real-world test?

## Block 2: Why this matters now
**Time:** 0:25-1:20

**On screen**
- Search/news/product screenshots or a simple trend board.
- Highlight the repeated phrase viewers keep seeing.

**Voiceover / talking head**
> This topic is everywhere right now, but most explanations skip the part that matters to a normal person: what would I actually do with it today?
>
> So the goal is not to make the most advanced version. The goal is to make the first useful version.

**Retention beat**
Make the viewer feel the pain of hype without practical next steps.

**Open loop**
What does the first useful version look like?

## Block 3: The simple mental model
**Time:** 1:20-2:20

**On screen**
- One diagram or checklist with 3-5 boxes.
- Use large labels and no dense paragraphs.

**Voiceover / talking head**
> Here is the mental model I am using for this video.
>
> First, what problem does it solve? Second, what proof can I show on screen? Third, what can go wrong? And fourth, would I use it again?
>
> If a tech trend cannot pass those four questions, it is probably not ready for your daily workflow.

**Retention beat**
Give viewers a reusable framework.

**Open loop**
Will this specific idea pass the framework?

## Block 4: Setup the demo
**Time:** 2:20-3:15

**On screen**
- Show the starting state.
- Show demo account/sample repo/sample files.
- Show version or settings screen if relevant.

**Voiceover / talking head**
> For the demo, I am keeping the setup intentionally small.
>
> Use a tiny repo task: add a CLI flag, update README, run tests, then ask each agent to fix one intentionally failing test.
>
> I am using sample data and demo accounts so nothing private ends up in the video.

**Retention beat**
Build trust by showing constraints and safety.

**Open loop**
Can the small demo produce a useful result?

## Block 5: Do the thing
**Time:** 3:15-5:30

**On screen**
- Screen recording of the actual workflow.
- Cut boring waiting time.
- Zoom on decisions, prompts, commands, settings, or output.

**Voiceover / talking head**
> This is the part I care about: not the marketing page, the workflow.
>
> I am looking for friction. How many steps does it take? Where do I get confused? Does the result match what I expected?
>
> If I have to babysit every step, that is not automatically bad, but it changes who I would recommend it to.

**Retention beat**
Use quick cuts every 20-30 seconds: setup, action, output, reaction.

**Open loop**
Where does the first problem appear?

## Block 6: Verification / scorecard
**Time:** 5:30-7:15

**On screen**
- Final result.
- Simple scorecard: setup friction, usefulness, risk, beginner fit.
- Any verification command/output or before/after comparison.

**Voiceover / talking head**
> Now I want proof, not vibes.
>
> Did it do the job? Could I explain the result? Could I undo or fix the mistake? And did it save enough time to justify adding another tool or habit?
>
> Here is the scorecard.

**Retention beat**
Resolve the main promise with a concrete verdict.

**Open loop**
What is the catch?

## Block 7: What worked and what did not
**Time:** 7:15-8:45

**On screen**
- Two-column graphic: `Worked` vs `Did not work`.
- Keep one real limitation in the edit.

**Voiceover / talking head**
> The good part is that this gives beginners a way to start without learning every detail first.
>
> The bad part is that the simple version can hide sharp edges.
>
> Tool behavior changes quickly; record exact versions and frame this as a beginner workflow test, not a permanent ranking.

**Retention beat**
Give honest tradeoffs so the video feels credible.

**Open loop**
So who should actually use this?

## Block 8: Recommendation
**Time:** 8:45-10:00

**On screen**
- Decision matrix: `Use this if...` / `Skip this if...`

**Voiceover / talking head**
> My recommendation: use this if you want a low-risk first step and you are willing to verify the result.
>
> Skip it if you need perfect reliability, private production data, or a tool you do not have time to understand.
>
> The beginner win is not doing the most advanced version. It is building one small habit you can repeat safely.

**Retention beat**
Give viewer decision clarity.

**Open loop**
What should they try next?

## Block 9: Wrap and next video
**Time:** 10:00-10:30

**On screen**
- Final checklist.
- Tease next related video.

**Voiceover / talking head**
> If you try this, start with a demo account, sample files, or a throwaway repo. Make the mistake cheap.
>
> Next, I want to test another beginner tech workflow the same way: small demo, real proof, honest verdict.
>
> If there is a tool or setup you want tested, comment it and I will put it on the board.

**Retention beat**
End with a clear next-step invitation, not a generic outro.
