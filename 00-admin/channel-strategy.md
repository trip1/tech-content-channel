# Channel Strategy

## Positioning

Practical, build-in-public tech content focused on software development, server infrastructure, homelabs, and real-world troubleshooting.

## Target audience

- Developers who want better infrastructure skills
- Homelab builders who want practical projects
- Tech-curious viewers who like seeing real systems built and debugged
- Newer engineers who benefit from transparent workflows and mistakes

## Content promise

Show the real process: decisions, tradeoffs, bugs, fixes, and working results.

## Voice/tone

- Practical and honest
- Curious, not guru-like
- Explain tradeoffs clearly
- Avoid hype unless the result earns it

## Recurring series ideas

| Series | Description | Format |
|---|---|---|
| Build the Lab | Homelab projects from idea to running service | stream + edited recap |
| Ship It Live | Build small tools/apps live | Twitch/Kick live |
| Infra Autopsy | Break down failures, outages, or debugging sessions | YouTube explainer |
| Dev Workflow Upgrades | Improve tooling, automation, editors, CI/CD | tutorial/review |
| Weekend Server Project | One useful server project per episode | short series |
